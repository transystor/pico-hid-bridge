# First build goal for firmware-c

Цель ближайшего этапа не в том, чтобы сразу получить полный TCP bridge, а в том, чтобы добраться до первого устойчивого milestones:

## Milestone 1
Собрать `.uf2`, который:
- стартует на Pico W,
- инициализирует TinyUSB,
- определяется на ПК как HID mouse,
- периодически делает маленькое demo-движение курсора,
- мигает onboard LED,
- пробует поднять Wi‑Fi,
- но ещё не зависит от готовности TCP listener.

## Почему это важно

Нам нужно развести риски по слоям:

1. **USB HID слой**
   - понять, что TinyUSB mouse report реально работает

2. **базовый runtime Pico SDK**
   - убедиться, что проект вообще собирается и стартует

3. **Wi‑Fi слой**
   - отдельно понять, как ведёт себя `cyw43_arch`

4. **TCP listener**
   - только после этого вешать поверх него сетевой протокол

## Что сейчас делает `main.c`

Текущий каркас в `firmware-c/src/main.c` уже ориентирован на такой первый milestone:
- поднимает `tusb_init()`
- мигает LED
- пытается подключиться к Wi‑Fi
- при готовом HID интерфейсе делает тестовое `mouse_report`

## Что ещё нужно будет доделать

- убрать compile-time заглушки `CHANGE_ME`
- оформить Wi‑Fi credentials через compile definitions или отдельный config header
- проверить корректность descriptor path на реальной сборке
- убедиться, что host действительно видит устройство как мышь
- только после этого добавлять lwIP TCP server
